# DNS (Domain Name System)

인터넷에서 사용되는 **도메인을 IP 주소로 변환**해주는 시스템

---

## 도메인 (Domain)

IP 주소를 사람이 기억하기 쉽도록 만드는 **문자 주소**

```
www.test.com
 │     │  └── 탑 레벨 도메인 (TLD)  예) .com
 │     └───── 도메인 이름
 └─────────── 서브 도메인
```

---

## DNS 구조

```
                                        ┌──────────────────┐
                                        │  Root DNS Server │
                                        └────────┬─────────┘
                                                 │
          ┌────────────┐    ┌───────────────┐    │    ┌──────────────────┐
          │ Web Server │←───│  ISP DNS Sever│←───┴───→│  TLD DNS Server  │
          └────────────┘    │(Local DNS Sev)│         └──────────────────┘
                     ↑      └───────┬───────┘
                     │              │
                  [User]            ↓
                            ┌──────────────────────┐
                            │ Authoritative DNS Svr│
                            └──────────────────────┘
```

### DNS 서버 종류

| # | 서버명 | 설명 |
|---|--------|------|
| 1 | **DNS 리졸버 (Resolver)** | 사용자로부터 최초 DNS 질의를 받는 DNS 서버. 질의 받은 도메인을 모른다면 DNS 계층 구조로 질의해 최종 IP를 알아온다. |
| 2 | **루트 네임서버 (Root)** | DNS 계층 구조의 최상단에 위치한 마스터 서버. TLD 네임서버에 대한 주소를 관리, 제공한다. |
| 3 | **TLD 네임서버 (Top-Level Domain)** | 최상위 도메인에 관한 정보를 관리하는 서버. 권한 있는 네임서버에 대한 주소를 관리, 제공한다. |
| 4 | **권한있는 네임서버** | 특정 도메인에 관한 직접적인 IP 정보를 관리하는 서버. DNS 리졸버의 최종 목적지가 된다. |

## DHCP (Dynamic Host Configuration Protocol)

**다이나믹 호스트 컨피그레이션 프로토콜**

- 엔드 디바이스 장치들에게 **네트워크 설정들을 자동으로 할당**해주는 시스템
- 많은 장치를 수동으로 관리해야 하는 **번거로움을 줄여**준다
- 사용하지 않는 장치의 IP 주소를 **회수**하여 한정된 IP를 **효율적으로 사용**할 수 있게 해준다
