## 1. 네트워크 공격 유형

### 1-1. HTTP Get Flooding 공격

> 서버에 전달되는 HTTP Get 패킷이 너무 많아서 다른 클라이언트가 해당 서버에 접속할 수 없게 하는 공격

**정상적인 과정**
- 사용자가 특정 URL에 접속 시 해당 웹 서버에 TCP 연결 후 HTTP Get 패킷을 전송
- 웹 서버는 해당 URL의 자료(HTML 문서 등)를 패킷에 넣어 응답

**공격 원리**
- 웹 서버는 매 순간 처리 가능한 HTTP 패킷의 양이 정해져 있음
- 공격자가 처리 용량을 초과하는 대량의 HTTP Get 패킷을 전송
- 웹 서버가 모든 시스템 자원을 소비하여 다른 패킷을 처리하지 못하는 상태가 됨
- 결과적으로 정상 사용자가 웹 페이지 요청 불가

**대응 방안**
- 방화벽에서 HTTP Get 패킷에 대한 **임계값(Threshold)** 을 설정하여 일정량 이상의 패킷을 차단

---

### 1-2. HTTP CC 공격

> HTTP Get 패킷에 캐싱 장비가 응답하지 않도록 설정하여 웹 서버 부하를 증가시키는 공격

**정상적인 과정**
- 캐싱 장비: 요청 정보가 자신에게 없으면 웹 서버로 전달, 있으면 자체 응답하여 웹 서버 부하 감소
- `Cache-Control: no-store` 또는 `must-revalidate` 설정 시 캐싱 장비가 응답하지 않고 웹 서버가 응답

**공격 원리**
- 공격자가 `Cache-Control`로 캐싱을 우회하는 HTTP Get 패킷을 대량으로 전송
- 캐싱 장비가 응답하지 않고 모두 웹 서버에 전달
- 웹 서버가 모든 시스템 자원을 소비하여 정상 서비스 불가

**대응 방안**
- 캐싱 장비 또는 그 앞단에 방화벽 설치
- 캐싱 우회 문자열이 포함된 패킷에 임계값을 설정하여 차단

### 1-3. DNS 질의 및 응답 과정

DNS 조회 순서:

```

3. 로컬 영역 DNS 서버
4. 로컬 영역 밖의 DNS 서버 (가입된 ISP 업체)
```

### 1-4. DNS Amplification 공격

> 공격자가 다수의 DNS 서버에 질의 패킷을 보내면서 특정 시스템에 응답 패킷이 집중되도록 조작하는 공격

**공격 원리 및 과정**

1. 공격자가 **출발지 IP를 공격 대상 서버의 IP로 위조**하여 다수의 DNS 서버에 질의 패킷 전송
2. 각 DNS 서버는 위조된 출발지 IP(= 공격 대상 서버)로 응답 패킷 전송
3. 대량의 응답 패킷이 대상 서버로 집중되어 네트워크 대역폭 초과
4. 다른 패킷의 전달이 지연되거나 누락 → 서버와 통신 불가

**대응 방안**
- 출발지 라우터에서 **IP Spoofing 검사**를 통해 출발지 IP가 변조된 패킷 차단
- 내부 네트워크 유입 DNS 응답 패킷에 임계값 설정

---

### 1-5. SIP Flooding 공격

> SIP PROXY 서버에 SIP INVITE request 패킷이 너무 많아서 해당 서버에 속한 시스템들과 통신할 수 없게 하는 공격

**정상적인 과정**
- 시스템 A → SIP INVITE request → SIP Proxy 서버 → 시스템 B
- 시스템 B → SIP INVITE response → 연결 수립

**공격 원리**
- 공격자가 다수의 SIP INVITE request 패킷을 발생시켜 전송
- 시스템 B의 자원을 모두 소비하거나 네트워크 대역폭 초과 → 정상 통신 불가

**대응 방안**
- 인증된 사용자만 SIP 연결 수립 가능하도록 설정
- 네트워크 유입 SIP 패킷에 임계값 설정하여 차단

---

### 1-6. UDP Flooding 공격

> 서버에 전달되는 UDP 패킷이 너무 많아서 다른 클라이언트가 해당 서버에 접속할 수 없게 하는 공격

**정상적인 과정**
- UDP는 TCP와 달리 연결 설정 없이 바로 데이터 전송 → 전송 속도 빠름
- 오류 발생 시 ICMP 등으로 보완

**공격 원리**
- 공격자가 대량의 UDP 패킷 전송 → 서버가 각 패킷을 처리할 응용 프로그램 검색
- 적절한 응용 프로그램이 없으면 서버가 **ICMP Unreachable** 패킷 응답 → 네트워크 부하 증가
- 시스템 자원 고갈 및 대역폭 초과 → 정상 서비스 불가

**대응 방안**
- Snort 또는 방화벽을 이용하여 임계값 이상의 UDP 패킷을 차단

---

### 1-7. ICMP Flooding 공격

> ICMP 패킷이 너무 많아서 처리 자원을 모두 소비하여 다른 시스템이 패킷을 전송할 수 없게 되는 공격

대표적인 공격: **Ping of Death**, **Smurf 공격**

#### Ping of Death

| 항목 | 내용 |
|------|------|
| 공격 방식 | 매우 큰 ICMP echo request 패킷을 대량으로 전송 |
| 패킷 크기 | 65,000 바이트 |
| 명령 옵션 | `ping -l` 또는 `hping3` 도구 사용 |
| 패킷 특징 | 큰 패킷이 네트워크에서 다수의 작은 패킷으로 분할 전송됨 |

**공격 과정**
1. 공격자가 출발지 IP를 랜덤하게 생성 후 특정 주소로 65,000바이트 패킷을 플러딩
2. 대상 시스템은 패킷을 재조립하고 ICMP echo reply를 전송해야 함
3. 자원 고갈 → 다른 패킷 처리 불가

**실습 환경 구성**
- 3대 이상의 PC를 공유기에 연결하거나
- VMware에서 3개 이상의 가상 머신을 Host-only로 연결
- 최신 OS는 다수의 ICMP 패킷을 무시하므로 Windows 95/98 또는 Linux 6.0 이하 사용 필요

#### Smurf 공격

**공격 원리**
1. 공격자가 ICMP echo request의 **출발지 IP를 공격 대상 시스템(B)의 IP로 위조**
2. 브로드캐스트 방식으로 다수의 에이전트에 전송
3. 각 에이전트가 ICMP echo reply를 시스템 B에 전송
4. 시스템 B의 자원 고갈 → 정상 통신 불가

**대응 방안**
- Snort 또는 방화벽을 이용하여 임계값 이상의 ICMP echo reply 패킷을 차단

## 2. 리눅스 기초

### 서버 vs 클라이언트

| 구분 | 설명 |
|------|------|
| **서버** | 기업용, 서비스 제공 |
| **클라이언트** | 일반용, 서비스 이용(요청) |

### 리눅스 배포판

```
리눅스
├── Debian 계열 → Ubuntu (무료)
└── RedHat 계열 → Rocky (무료)
```

### 인터페이스 방식

| 약어 | 풀네임 | 설명 |
|------|--------|------|
| GUI | Graphical User Interface | 그래픽 모드 (키보드, 마우스) |
| TUI | Text User Interface | 텍스트 모드 |
| CLI | Command Line Interface | 명령문 모드 |

### 기본 개념

| 개념 | 설명 |
|------|------|
| `/` | 최상위 디렉터리 및 리눅스 경로 표시 |
| `IP` | 인터넷 프로토콜, 인터넷 통신 수단 |
| SSH | 원격 서버 접속 프로토콜 |
| Xshell | 원격 접속 도구 |

### 명령 프롬프트

```bash
$   # 일반 사용자 프롬프트
#   # 관리자(root) 프롬프트
```

### 관리자 권한 전환

```bash
# 1단계: 관리자 비밀번호 설정
sudo passwd root

# 2단계: 관리자 전환
su -
```

### 디렉터리 구조

| 리눅스 경로 | Windows 대응 경로 | 설명 |
|-------------|-------------------|------|
| `/` | `C:\` | 최상위 디렉터리 |
| `/home/` | `C:\Users\` | 사용자 홈 디렉터리 기본 경로 |
| `/root/` | — | 관리자 홈 디렉터리 |
| `/etc/` | `C:\Windows\System32\` | 시스템 관련 설정 파일 |
| `/bin` → `/usr/bin/` | — | 기본 명령어 실행 파일 |
| `/sbin` → `/usr/sbin/` | — | 시스템 명령어 (관리자 전용) |

## 3. 기본 명령어

### 위치 및 이동

```bash
pwd                  # 현재 위치(경로) 확인
cd <경로대상>         # 디렉터리 이동

# 경로 표기법
.                    # 현재 경로
..                   # 상위 경로
/                    # 절대 경로 (최상위부터 시작)
./samadal            # 상대 경로 (현재 위치에서 시작)
cd ./../             # 현재 위치에서 상위 디렉터리로 이동
```

### 목록 확인 (ls)

```bash
ls                   # 기본 목록 나열
ls -a                # 숨김 파일 포함 전체 나열 (. 으로 시작하는 파일)
ls -l                # 속성 정보 자세히 보기
ls -R /home/test/    # 하위 디렉터리까지 재귀적으로 나열
ls -ld <경로>        # 디렉터리 자체의 속성 확인
```

**ls -l 출력 형식**

```
d rwxr-xr-x  3  root root  4096  9월 8 20:35  dirname
│ └────────┘  │  └──────┘  └──┘  └────────┘
│  권한(허가권)  링크수  소유자/그룹  크기  날짜
└── d: 디렉터리, -: 파일
```

### 파일 내용 출력

```bash
cat /etc/passwd              # 파일 전체 내용 출력
head /etc/passwd             # 위에서 10줄 출력 (기본값)
tail /etc/passwd             # 아래에서 10줄 출력 (기본값)
head -2 /etc/passwd          # 2줄만 출력
tail -15 /etc/passwd         # 15줄 출력
nl /etc/passwd               # 줄 번호 붙여서 출력
grep root /etc/passwd        # 특정 문자열 포함 줄만 출력
```

### 파이프라인 (|)

> A | B : A 명령의 결과를 B 명령의 입력으로 전달

```bash
ls -l /etc/ | nl                          # 목록에 줄 번호 붙이기
ls -l /etc/ | head -3                     # 상위 3줄만 출력
ls -l /etc/ | tail -5 | nl               # 하위 5줄에 줄 번호 붙이기
cat /etc/passwd | grep samadal | nl       # 검색 후 줄 번호 표시
ifconfig | head -2                        # IP 정보 상위 2줄 확인
```

### 출력 리다이렉션 (>, >>)

```bash
# > : 덮어쓰기 (강제 생성)
cat > file1              # 입력 후 Ctrl+D 로 저장

# >> : 이어쓰기 (내용 추가)
ls -l /etc/ | head -5 | nl >> file2
cat file1 file2 >> file3
```

### 파일 검색 (find)

```bash
find / -name "samadal"            # / 부터 "samadal" 이름 검색
find / -name "*samadal" | nl      # samadal로 끝나는 모든 것 (개수 확인)
find / -name "*samadal" -type d   # 디렉터리만 검색
find / -name "*samadal" -type f   # 파일만 검색
```

| 타입 옵션 | 의미 |
|-----------|------|
| `-type d` | 디렉터리 |
| `-type f` | 파일 |
| `-type l` | 링크 파일 |
| `-type b` | 블록 장치 파일 |
| `-type c` | 캐릭터 장치 파일 |

### 와일드카드

| 기호 | 의미 |
|------|------|
| `*` | 모든 글자 대체 |
| `?` | 글자 하나만 대체 |
| `*cfg` | cfg로 끝나는 모든 것 |
| `Login*` | Login으로 시작하는 모든 것 |
| `*conf*` | conf가 포함된 모든 것 |

### 기타 유용한 명령어

```bash
touch <파일명>           # 빈 파일 생성 (이미 있으면 타임스탬프만 갱신)
```

## 4. 파일 조작 명령어

### mkdir (디렉터리 생성)

```bash
mkdir ./a/                           # a 디렉터리 생성
mkdir -p /home/samadal/a/b/e/f/g/   # 중간 경로 포함 일괄 생성
# -p (parents): 필요한 부모 디렉터리도 함께 생성
```

### cp (복사)

```bash
# 파일 복사 (이름 유지)
cp /backup/grub.cfg ./

# 파일 복사 (이름 변경)
cp /backup/debconf.conf ./debconf

# 여러 파일 복사
cp <원본1> <원본2> ... <목적지>

# 디렉터리 복사
cp -r /backup/ ../test/a/    # -r: 디렉터리 및 내용물 포함

# 속성값 유지 복사
cp -p /backup/* ./           # -p: 권한, 소유자, 타임스탬프 유지
```

> ⚠️ **주의:** 여러 개 복사 시 이름 변경 불가 / 원본은 건드리지 않음

### mv (이동 및 이름 변경)

```bash
# 파일 이동
mv ./a/backup/ /

# 이름 변경 (cp + 삭제 개념)
mv /backup/grub.cfg /backup/grub.conf

# 여러 파일 이동
mv ./c/d* ./c/g* ./c/l* ./c/backup
```

> ⚠️ **주의:** 스페이스바 하나로 원본/목적지가 구분되므로 주의

### rm (삭제)

```bash
rm -rf grub.cfg             # 파일 삭제
rm -rf ./a/                 # 디렉터리 삭제
rm -rf /backup/*            # 디렉터리 내용물만 삭제
rm -rf /backup/             # 디렉터리 자체 삭제
```

| 옵션 | 의미 |
|------|------|
| `-r` | 디렉터리 및 내용물 포함 |
| `-f` | 확인 메시지 없이 강제 삭제 |

> ⚠️ **주의:** 한번 삭제하면 복구 불가능! 삭제 후 반드시 확인할 것

## 6. 사용자 및 그룹 관리

### 관련 파일

| 파일 경로 | 내용 |
|-----------|------|
| `/etc/passwd` | 사용자 계정 정보 |
| `/etc/shadow` | 사용자 비밀번호 (암호화) |
| `/etc/group` | 그룹 계정 정보 |

### /etc/passwd 구조

```
samadal : x : 1000 : 1000 : samadal,,, : /home/samadal : /bin/bash
   │      │     │      │        │              │               │
  계정명  비번  UID   GID    Comment        홈디렉터리         쉘
```

| 비밀번호 필드 | 의미 |
|--------------|------|
| `x` | 비밀번호 설정 확인 → `/etc/shadow` 에서 확인 |
| `*` | 비밀번호 설정 불가능한 계정 |
| `!` | 비밀번호 미설정 계정 |

---

### 사용자 생성 (useradd)

```bash
# 기본 (홈디렉터리 미생성 - 비권장)
useradd user1

# 홈디렉터리 자동 생성 (권장)
useradd -m user2

# 닉네임, 쉘 지정
useradd -c ubuntuuser -s /bin/bash -m user3

# 홈디렉터리 경로 변경
mkdir /cloud/
useradd -m -d /cloud/user4 user4
```

**계정 작업 순서**

```bash
# 1. 계정 생성
useradd -m user1

# 2. 계정 정보 확인
tail -4 /etc/passwd

# 3. 홈디렉터리 확인
ls -l /home/

# 4. 비밀번호 설정
passwd user1
```

**skel 기능**
- `/etc/skel/` 안의 파일/디렉터리를 미리 생성해두면
- 사용자 생성 시 홈 디렉터리에 자동 복사됨

### 사용자 수정 (usermod)

```bash
usermod -g user1 -s /bin/bash -c testuser user2

# 홈디렉터리 변경 (2단계 작업)
usermod -d /export/home/user2 user2     # 1단계: 경로 정보 변경
mv /home/user2/ /export/home/           # 2단계: 실제 디렉터리 이동
```

| 옵션 | 의미 |
|------|------|
| `-g` | 그룹 변경 |
| `-s` | 쉘 변경 |
| `-c` | 닉네임(Comment) 변경 |
| `-d` | 홈디렉터리 경로 정보 변경 |
| `-md` | 홈디렉터리 정보 변경 + 실제 이동 (정상 계정에만 가능) |
| `-G` | 그룹 추가 |

> ⚠️ **주의:** 사용자가 접속 중인 상태에서는 홈디렉터리 변경 불가

---

### 사용자 삭제 (userdel)

```bash
userdel -r <계정명>    # 계정 및 관련 파일 모두 삭제
```

### 그룹 관리

```bash
# 그룹 생성
groupadd g

# 그룹 변경 (GID 변경)

# 그룹 삭제
groupdel g

# 계정 정보 확인

# 출력: uid=1001(user1) gid=1001(user1) groups=1001(user1)

```

## 7. 권한 관리

### 권한 구조

```
U (User) / G (Group) / O (Others)

d  rwx  r-x  r-x
│  └─┘  └─┘  └─┘
│  소유자 그룹  기타
└── 파일 유형 (d: 디렉터리, -: 파일)
```

### 권한 값

| 권한 | 기호 | 값 | 파일 의미 | 디렉터리 의미 |
|------|------|----|-----------|---------------|
| Read | `r` | 4 | 파일 내용 읽기 | 디렉터리 내 목록 확인 |
| Write | `w` | 2 | 파일 내용 수정 | 파일 생성/삭제 |
| Execute | `x` | 1 | 파일 실행 | 디렉터리 접근(진입) |

### 허가권 변경 (chmod)

```bash
# 숫자 방식
chmod 623 login.defs

# 문자 방식
chmod u+r,u+w,u-x,g-r,g+w,g-x,o-r,o+w,o+x login.defs
```

### 소유권 변경 (chown)

```bash
chown samadal grub.cfg           # UID만 변경
chown :samadal debconf.conf      # GID만 변경
chown root:samadal grub.cfg      # UID, GID 모두 변경
chown samadal: login.defs        # UID와 GID 동일하게 변경
```

> ⚠️ **권한 변경 명령은 반드시 root(관리자) 권한으로 실행**

## 8. 압축 및 아카이브

### 압축 도구 비교

| 도구 | 속도 | 압축률 | 확장자 |
|------|------|--------|--------|
| `gzip` | 빠름 | 보통 | `.gz` |
| `bzip2` | 느림 | 높음 | `.bz2` |
| `xz` | 느림 | 가장 높음 | `.xz` |
| `zip` | 보통 | 보통 | `.zip` |

### 압축 명령어

```bash
# gzip
gzip <파일>             # 압축
gunzip <파일.gz>        # 해제

# bzip2
bzip2 <파일>            # 압축
bunzip2 <파일.bz2>      # 해제

# xz
xz <파일>               # 압축
unxz <파일.xz>          # 해제

# zip
zip <출력.zip> <파일>   # 압축
unzip <파일.zip>        # 해제 (현재 위치에 풀림)
```

> ⚠️ **주의:** 압축 대상은 '파일만', '1개만' 가능 → 여러 파일 묶으려면 **tar** 사용

---

### tar (아카이브)

> 여러 파일/디렉터리를 하나의 파일로 묶음

**tar 옵션**

| 옵션 | 의미 |
|------|------|
| `c` | 아카이브 생성 (묶기) |
| `x` | 아카이브 풀기 |
| `v` | 작업 과정 상세 출력 |
| `f` | 아카이브 파일 이름 지정 |
| `z` | gzip 연동 (`*.tar.gz`) |
| `j` | bzip2 연동 (`*.tar.bz2`) |
| `J` | xz 연동 (`*.tar.xz`) |

```bash
# 아카이브 생성
tar cvf dgl.tar grub.cfg debconf.conf login.defs

# 아카이브 풀기
tar xvf dgl.tar

# tar + gzip 압축
tar cvfz /export/home/dglg.tar.gz *

# tar + gzip 해제
tar xvfz dglg.tar.gz
```

## 10. HDD 관리

### HDD 작업 순서

```
1. 하드 디스크 추가 (SCSI 방식)
2. HDD 이름 확인
3. 파티션 작업 (fdisk)
4. 포맷 (mkfs)
5. 마운트 (mount)
```

### HDD 이름 확인

```bash
fdisk -l <hdd 장치명>
```

### 파티션 작업 (fdisk)

```bash
fdisk /dev/sdb
```

| 명령 | 동작 |
|------|------|
| `d` | 파티션 삭제 |
| `n` | 파티션 추가 |
| `p` | 파티션 테이블 확인 |
| `q` | 저장 없이 종료 |
| `w` | 저장 후 종료 |

### 포맷 (mkfs)

```bash
sudo mkfs.ext4 /dev/sdb1
# 또는
sudo mkfs -t ext4 /dev/sdb1
```

### 마운트 (mount)

```bash
# 마운트 연동
sudo mount /dev/sdb1 /mp/

# 마운트 확인
sudo df -h

# 마운트 해제
sudo umount /dev/sdb1
# 또는
sudo umount /mp/
```

> M.P(Mount Point) : 사용되지 않는 빈 디렉터리를 HDD 장치와 연동하는 디렉터리

### 마운트 유의사항

1. M.P는 **사용되지 않는** 임의의 디렉터리여야 함
2. mount/umount 명령 시 현재 경로가 M.P 디렉터리여서는 안 됨 (사용 중 변경 불가)
3. 하나의 파티션은 반드시 **하나의 M.P** 만 사용 (1:1 대응)
4. 포맷 및 마운트 대상은 **주 파티션(P), 논리 파티션(L)** 만 가능 (확장 파티션(E) 불가)
5. 파티션 작업 시 반드시 마운트 연동을 해제하고 진행
