# TCP/IP 4 Layer

- 실질적인 **인터넷 프로토콜 구현**을 목적으로 만들어진 모델
- OSI 7 Layer보다 **간단**하며, **현대 네트워크의 표준**이 된다

---

## OSI 7계층 vs TCP/IP 4계층 대응표

```
OSI 7 Layer                프로토콜 예시       TCP/IP 4 Layer
─────────────────────────────────────────────────────────────
7계층  응용 계층             ┐
       Application Layer    │  HTTP, FTP,     응용 계층
6계층  표현 계층             ├─ DNS, SSH  ───  Application Layer
       Presentation Layer   │
5계층  세션 계층             ┘
       Session Layer
─────────────────────────────────────────────────────────────
4계층  전송 계층                              전송 계층
       Transport Layer    ──  TCP, UDP  ───  Transport Layer
─────────────────────────────────────────────────────────────
3계층  네트워크 계층                           인터넷 계층
       Network Layer      ──  IP, ICMP  ───  Internet Layer
─────────────────────────────────────────────────────────────
2계층  데이터 링크 계층        ┐               네트워크 액세스 계층
       Data Link Layer     ├─ Ethernet, ──  Network Access Layer
1계층  물리 계층              ┘ MAC Address
       Physical Layer
─────────────────────────────────────────────────────────────
```

---

## 계층별 설명

| # | 계층명 | 설명 |
|---|--------|------|
| 1 | **네트워크 액세스 계층** (Network Access Layer) | 데이터 전송의 **물리적 연결**과 관련된 계층. 케이블 등의 물리적 매체를 통하여 실제 데이터를 신뢰성 있게 전송한다. **MAC 주소**를 통해 정확한 장치로 데이터가 전송되도록 보장한다. |
| 2 | **인터넷 계층** (Internet Layer) | 데이터가 올바른 목적지로 전송되도록 **라우팅**을 하는 계층. **IP**를 통하여 데이터의 송/수신지를 정의한다. |
| 3 | **전송 계층** (Transport Layer) | **종단간 통신**에서의 신뢰성을 보장하는 계층. **Port 번호**를 통하여 장치 내의 애플리케이션, 서비스, 프로토콜을 식별한다. |
| 4 | **응용 계층** (Application Layer) | 최종 사용자에게 **서비스를 제공**하는 계층. 네트워크를 통해 데이터를 주고 받는 응용프로그램들이 동작한다. |
